Реализация утилиты `grep` на Go с поддержкой контекста и базовых опций.

## Синтаксис
grep [OPTIONS] PATTERN [FILE...]

text

Если FILE не указан или указан `-`, читается стандартный ввод.

## Опции

| Опция | Описание |
|-------|----------|
| `-A N` | Печатать `N` строк после совпадения |
| `-B N` | Печатать `N` строк до совпадения |
| `-C N` | Печатать `N` строк до и после совпадения (контекст) |
| `-c`   | Подсчитать количество совпадающих строк (вместо вывода) |
| `-i`   | Игнорировать регистр при сравнении |
| `-v`   | Инвертировать совпадение (выводить строки, НЕ соответствующие шаблону) |
| `-F`   | Точное совпадение строки (фиксированная строка, не регулярное выражение) |
| `-n`   | Печатать номера строк (начиная с 1) |

## Особенности

- Если шаблон пустой (например, не указан или равен пустой строке), совпадают все строки.
- При использовании `-F` шаблон интерпретируется как фиксированная подстрока (с опцией `-i` – без учёта регистра).
- Группы контекста разделяются строкой `--`.
- Опции `-A`, `-B`, `-C` могут быть заданы как слитно с числом (`-A5`), так и раздельно (`-A 5`).
- Поддерживается одновременное указание нескольких флагов без аргументов (например, `-ivn`).

## Примеры

1. Поиск строки "error" в файле `log.txt` с выводом 2 строк до и после:
grep -C2 error log.txt

text

2. Подсчёт количества строк, содержащих "panic" без учёта регистра:
grep -ci panic log.txt

text

3. Вывод всех строк, не содержащих "debug", с номерами:
grep -vn debug log.txt

text

4. Точное совпадение слова "INFO" (без учёта регистра):
grep -Fi INFO log.txt

text

5. Чтение из стандартного ввода:
cat log.txt | grep -A1 warning

text

6. Несколько файлов:
grep -n TODO *.go

text

## Возвращаемое значение

Программа завершается с кодом `0`, если ошибок не возникло (даже если совпадений не найдено). При ошибках (неверные флаги, проблемы с чтением файлов) возвращается `1` и сообщение выводится в stderr.
